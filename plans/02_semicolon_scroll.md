# 02. セミコロンキー（;）長押しによるスクロールレイヤー遷移の設定

セミコロンキー（`;`）を長押し（ホールド）した際に、OS環境に合わせたスクロールレイヤー（Macなら `Mouse_Scroll_Mac`、Winなら `Mouse_Scroll_Win`）に遷移する設定を実装しました。
タイピング時の誤爆を防ぐため、こちらも `require-prior-idle-ms` を備えたカスタムレイヤータップ（`lt_idle`）を新たに定義して適用しています。

---

## 課題
* トラックボール操作とスクロール操作の切り替えをよりスムーズにするため、右手のホームポジションにほど近いセミコロンキー（`;`）を長押しすることで即座にスクロール操作に入りたいという要望がありました。
* 同時に、タイピング中に `;` キーを打つ際に誤ってスクロールレイヤーに遷移（誤爆）してしまうことを防ぐ必要があります。
* さらに、オートマウスモード（トラックボールを動かしたときに一時的に遷移する `mouse` レイヤー）に入っている間でも、同様にセミコロン長押しでスクロール機能にアクセスできるように設計する必要がありました。

---

## 解決策

### 1. カスタムレイヤータップ `lt_idle` の追加
修飾キーではなく「レイヤー切り替え」に誤爆防止（`require-prior-idle-ms = <125>;`）を適用するため、新しい hold-tap ビヘイビア `lt_idle` を定義しました。

ホールド時のアクションにレイヤーモーメンタリー遷移（`&mo`）を指定しています。

```dts
    behaviors {
        // ... mt_idle ...

        lt_idle: lt_idle {
            compatible = "zmk,behavior-hold-tap";
            #binding-cells = <2>;
            flavor = "balanced";
            tapping-term-ms = <200>;
            quick-tap-ms = <0>;
            require-prior-idle-ms = <125>;
            bindings = <&mo>, <&kp>;
        };
    };
```

### 2. キーマップの書き換え

* **`Mac` レイヤー (Layer 0)**:
  セミコロンキーを `&lt_idle 6 SEMICOLON` に変更し、ホールド時に `Mouse_Scroll_Mac`（Layer 6）へ、タップ時に `;` を入力するように設定しました。
* **`Win` レイヤー (Layer 1)**:
  セミコロンに対応する透過位置（右から2番目のキー）を `&lt_idle 7 SEMICOLON` に変更し、ホールド時に `Mouse_Scroll_Win`（Layer 7）へ、タップ時に `;` を入力するように設定しました。

### 3. オートマウスレイヤー（Layer 5 `mouse`）との連携（透過設定）
オートマウスレイヤーにおけるセミコロンキーの位置は、元のデフォルトレイヤーの挙動を引き継ぐ **`&trans` (透過)** のままとしました。

これにより、トラックボールを動かして一時的にオートマウスレイヤーにいる状態でも、下位レイヤーの設定が透過的に引き継がれるため、自動的に現在のOS環境に応じたスクロール遷移（Macなら Layer 6, Winなら Layer 7）が呼び出されます。

---

## 効果と検証
* 通常打鍵時は `;` が軽快に入力でき、スクロールが暴発することはありません。
* スクロールしたい時は、`;` を意識して一瞬空けて長押しすることで、トラックボールを動かしたときにスクロールができるようになり、非常に直感的な片手操作が可能になりました。
* マウススクロールレイヤー（Layer 6, 7）自体はすべて `&trans` 設定となっており、透過の仕組みを通じて他の誤爆防止キー（`a`, `z` などの `mt_idle`）も問題なく機能します。
